# 基盤理論 — パラダイム中立な数学的基礎

> contingency-dsl 基盤層の一部。オペラント（三項随伴性）とレスポンデント（二項随伴性）が共有する、パラダイム中立な数学的基礎を定義する。すなわち、随伴性の一般的定義、文法の決定性、および非チューリング完全（non-TC）主張である。オペラント固有の導出（三項随伴性計算、スケジュール代数、コンビネータ意味論）は `operant/theory.md` に、レスポンデント固有の導出は `respondent/theory.md` に置く。

---

## 1. 一般関係としての随伴性

実験行動分析において、**随伴性**（contingency）とは事象間の確率的／論理的関係である。基盤層はこの関係を抽象的に定義し、オペラントおよびレスポンデントの各層が特殊化する。

**定義 1（事象）.** 事象とは時点上の発生であり、`stimulus-typing.md` および `time-scales.md` で定義される以下のカテゴリのいずれかに属する。

- 刺激の onset / offset（CS, US, SD, SΔ）
- 反応（R）
- 結果（Sr+, Sr−, 罰子）
- 時刻刻み（クロック進行）

**定義 2（随伴性）.** 随伴性とは三つ組 `(A, B, P)` であり、`A` と `B` は事象型、`P` はそれらを関連付ける条件付き確率構造である。標準的な特殊化は 2 種類存在する。

- **二項随伴性**（レスポンデント）: `P(US | CS)` と `P(US | ¬CS)` — パヴロフ型随伴性空間（Rescorla, 1967）。詳細は `contingency-types.md` および `respondent/theory.md`。
- **三項随伴性**（オペラント）: `P(Sr+ | R, SD)` — スキナー型の三項随伴性（SD–R–SR）。詳細は `contingency-types.md` および `operant/theory.md`。

随伴的配置と非随伴的（反応非依存）配置の区別は基盤的な概念であり、`contingency-types.md` において独立に扱う。

## 2. 文法の決定性

**命題 1（構文的決定性）.** 有効な任意の入力文字列 `s` に対して、DSL 文法はただ 1 つの構文木を許容する。これは `ll2-proof.md` の LL(2) 証明により確立される。オペラント合成 `arg_list` 内部の 1 か所のローカル LL(2) 決定点を除き、すべての決定点は LL(1) であり、どの決定点も複数の選択肢を許容しない。

**命題 2（意味的決定性）.** 構文木と標準初期状態 `σ₀` が与えられたとき、意味解析フェーズは一意な解決済み AST を生成する。3 フェーズ・パイプライン（`foundations/grammar.md` §4）は全域である。各フェーズ遷移は、次のフェーズの AST を生成する完全な成功か、AST を返さずエラーリストを報告する完全な失敗かのいずれかである。いずれのフェーズも非決定性を導入しない。

**系.** `parse ∘ resolve` はソース文字列から解決済み AST への部分関数である。成功時は出力が一意であり、失敗時はエラーリストが生成される。

## 3. 非チューリング完全（non-TC）主張

**主張.** 基盤・オペラント・レスポンデント・合成・実験の各層を包含する DSL は、**チューリング完全ではない**。

**根拠.** 各層は `architecture.md §4.1.1` で定義されるスケジュール式不変性（Schedule Expression Invariance, SEI）の性質 P1, P2, P3 により特徴付けられる。

- **P1（式木の固定性）.** AST は解析時に完全に決定され、セッション中に構造的変化を起こさない。
- **P2（パラメータの列挙可能性）.** すべてのパラメータは、リテラル値か、あるいは閉形式で解析時に固定された規則の出力である。
- **P3（単一スケジュール同一性）.** スケジュールの同一性はフェーズ内で不変であり、ランタイム状態に基づいて構造的に異なる式に遷移することはない。

チューリング完全な実行時機能（実行時観測量による任意条件分岐、再帰的スケジュール遷移、被験体間ヨーク評価、外部状態に条件付けられたパラメータ更新）は、姉妹パッケージ（`contingency-core`, `experiment-core`）に委譲される。層境界の全体像は `architecture.md §4.1` を参照。

**これにより得られるもの.** 非 TC 文法は、TC 言語では表現できない静的に決定可能な性質を許容する。

- 到達可能性（「この手続きは強化子を生成しうるか?」）
- デッドコード検出（「全コンポーネントは到達可能か?」）
- 構造的妥当性（「`Conc` は 2 個以上のコンポーネントを持つか?」）
- 木構造に紐付いたリンター警告（例: オペラント層の並行スケジュールにおける `MISSING_COD`）

これらの性質は、Chomsky 階層における文法の位置が CFG + 有界マクロ展開（`let`, `Repeat`）であるため、構文木の走査により決定可能となる。

## 4. 3 つの等価関係

すべてのパラダイム層は、式に関する以下の関係を共有する。パラダイム固有の精緻化（観測トレース上で動作する意味的等価性 `≡`）は、各層の `theory.md` に置く。

**定義 3（構文的等号 `=_syn`）.** 2 つの式 `E₁` と `E₂` が *構文的に等しい* とは、それらの AST 表現が構造的に同一であること、すなわちすべてのノード型、演算子、パラメータ値、子順序が一致することである。構文的等号は再帰的な構造比較により決定可能である。

**定義 4（意味的等価性 `≡`）.** 2 つの式が *意味的に等価* とは、あらゆる可能な観測トレースに対して同一の結果を生成することである（トレースの宇宙はパラダイム固有である。オペラントは時刻刻み付きの反応ストリーム、レスポンデントは時刻刻み付きの CS/US 事象ストリームを用いる）。否定は `≢` と書く。

**定義 5（健全な書換規則 `→`）.** *健全な書換規則* `E₁ → E₂` とは、`E₁ ≡ E₂` となる有向 AST 変換である。適合実装は健全な書換を最適化パスとして適用してよい（MAY）が、健全でない書換を適用してはならない（MUST NOT）。

オペラント層とレスポンデント層はそれぞれ、これらの関係を具体的なトレース宇宙と結果列の定義で具象化する。

## 5. 型安全性（パラダイム中立）

整形された DSL プログラムは、意味解析中に型エラーを発生させない。論証は Pierce（2002, TAPL §8.3）を、DSL の 3 フェーズ・コンパイルパイプラインに適合させて踏襲する。パラダイム中立の主張は次の通りである。

**定理（進行 + 保存、基盤層の言明）.** 任意のパラダイム層において成功裏に構文解析される任意のソース文字列 `s` に対して、

- （*進行*）AST はフェーズ 3 にある（値である）か、もしくは以下のいずれか 1 つにより還元可能である: 束縛展開、`Repeat` 糖衣展開（そのパラダイム層に適用可能な場合）、パラダイム固有のデフォルト伝播。
- （*保存*）各フェーズ遷移は整型性を保存する。すなわち、整型のフェーズ k AST を整型のフェーズ k+1 AST へ写すか、型付きエラーリストで失敗する。

**系（型エラーなし）.** 整形された DSL プログラムは意味解析中に型エラーを生成しない。全エラーは、語彙的、構文的、またはパラダイム固有の意味的エラー（エラーコード付きで報告される制約違反）のいずれかである。

この定理はオペラント層について `operant/theory.md §2.12` で、レスポンデント層について `respondent/theory.md` で証明される。論証構造は同一であり、再利用のためここで抽象化している。

## 6. スコープの限界

この基盤理論は DSL の静的意味論（コンパイルパイプライン）を扱う。ランタイム正当性 — 解決済み AST がスケジュールエンジンによって解釈されるときに意図された振る舞いを生成すること — はエンジン実装（例: `contingency-core`）に依存し、基盤層のスコープ外である。
